# Guided Learning Mentor — 改进设计

## 背景

当前 `guided-learning-mentor` 技能采用 4 步 Socratic 学习循环，存在四个待改进点。

## 改动清单

### 1. 退出条件（新增 Step 5）

**问题**：学习循环没有明确的退出机制，学到什么程度算结束不清晰。

**方案**：三个退出信号 + 主动提议 + 用户确认。

- **信号 1 — 理解验证通过**：Step 3 中用户正确回答理解检查问题，且无追问
- **信号 2 — 追问枯竭**：同一话题连续 2 轮只有确认性回复（"懂了""明白了"），无新问题
- **信号 3 — 话题闭环**：若开始时机明确学习目标（"搞懂 X"），X 的子话题走完时自然收束

**收尾动作**：
1. 2-3 句核心要点回顾
2. 如有可记的关键规则/公式，列出
3. 问："想深入相关话题，还是先到这里？"

收尾是提议而非单方面结束，用户决定是否继续。

### 2. 字数限制（软性化）

**问题**：硬性 200 字上限迫使连贯概念被切碎，打断理解流。

**方案**：改为软性建议，核心原则 **"宁整不碎"**。

```
字数指引：目标 100-200 字/条。如果一个概念需要 300-400 字才完整，不要切碎——宁整不碎。
用户更在意理解的连贯性。
```

### 3. 套路可记 → 关键要点回顾

**问题**：当前写明 "Always include memorizable patterns"，但并非所有话题都有可记的公式或规则，硬挤显得生硬。

**方案**：去掉强制要求，改为收尾时的自然总结。

```
在收尾时，如果话题有值得记住的规则、公式、步骤，用简洁的列表呈现。
没有就不硬列——由智能体自行判断。
```

### 4. 飞书文档 token 解耦

**问题**：doc token 硬编码为固定值，降低技能可复用性。

**方案**：首次检测 + 引导配置。

- 技能加载时检查 `.claude/settings.local.json` 中是否有飞书 token
- 无配置则引导：
  > "我支持把学习笔记同步到飞书文档。要绑定吗？只需提供你的飞书文档链接（或 token），之后每次学习结束我都可以帮你自动归档要点。"
- 用户提供后写入 `.claude/settings.local.json`
- 后续行为和当前一致，但使用配置中的 token

## 需修改的文件

- `.claude/skills/guided-learning-mentor/SKILL.md` — 所有改动集中在该文件
