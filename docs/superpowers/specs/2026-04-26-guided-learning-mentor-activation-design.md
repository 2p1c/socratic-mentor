# Guided Learning Mentor — 激活与退出机制设计

## 背景

当前技能依赖项目 CLAUDE.md 声明来激活，缺少意图检测和退出机制。需要设计一套完整的激活/退出规则，支持全局安装场景和代码仓库学习场景。

## 设计

### 1. 两级激活

**Level 1 — 环境激活**（技能加载时）：

```
1. 检查 .claude/settings.local.json → 已有 guided-learning-mentor.activated = true？
   └─ 是 → 直接待命，跳过后续步骤

2. 当前目录有代码文件（排除 .claude/ .git/ docs/ 等配置目录）？
   ├─ 是 → 问："这个仓库里有代码，你想学习这个仓库的内容吗？"
   │       ├─ 是 → 调用 code-explorer 了解仓库结构 → 进入代码学习模式
   │       └─ 否 → 进入纯问答学习模式
   └─ 否（空目录）→ 进入纯问答学习模式

3. 首次被意图激活后，问：
   "要不要把学习模式设为这个项目的默认？下次进来就不用重复确认了。"
   └─ 是 → 写入 .claude/settings.local.json
```

**Level 2 — 意图激活**（用户发消息时）：

| 用户意图 | 行为 |
|---------|------|
| 学习类请求（"讲讲""怎么理解""为什么""是什么"） | 进入 Socratic 循环 |
| 模糊请求 | **默认学习**——先假设用户想学，引导而非直接做 |
| 明确"直接帮我做""别问了""直接给答案" | 退出学习模式，正常工程对话 |

### 2. 退出条件（三层防护）

| 层级 | 触发条件 | 示例 |
|------|---------|------|
| 用户明确表达 | "直接做""别问我""给我答案""别引导" | 正常工程模式 |
| 纯工程/元指令 | 无学习成分的工程操作 | git 操作、改技能配置、settings 管理 |
| 意图检测不匹配 | 连续消息都不含学习意图 | 返回静默待命 |

退出后用户随时可以说"学习模式"重新进入。

### 3. 两种学习模式

| | 纯问答模式 | 代码学习模式 |
|------|---------|---------|
| 上下文来源 | 用户提问驱动 | 仓库代码 + 用户提问 |
| 教学方式 | Socratic 对话 | Socratic 对话 + 引用实际文件/行号 |
| 辅助工具 | 无 | code-explorer 追踪代码 |
| 特色互动 | — | "从这个入口跟下去，你觉得下一步会调到哪个函数？" |

代码学习模式下，回答中引用实际代码位置（`src/auth/middleware.ts:42`），穿插读代码练习。

### 4. 全局安装安全边界

- 技能可以装在任何项目，默认静默
- 不依赖 CLAUDE.md 声明即可正常工作
- 意图检测 + 退出条件保证不会误切学习模式

## 需修改的文件

- `.claude/skills/guided-learning-mentor/SKILL.md` — 新增环境激活、意图检测、退出条件、代码学习模式
